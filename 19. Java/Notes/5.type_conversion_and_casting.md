# 5. Type Conversion & Casting

## Type Conversion (Automatic / Widening)
- Small type --> Big type = **automatic** (no data loss)
- `byte -> short -> int -> long -> float -> double`

```java
class Hello {
    public static void main(String args[]) {

        // Automatic conversion - small to big
        byte b = 10;
        int num = b;       // byte automatically converts to int
        System.out.println(num); // 10

        int i = 100;
        double d = i;     // int automatically converts to double
        System.out.println(d);   // 100.0

        // Type Promotion in expressions
        byte a = 10;
        byte c = 30;
        int result = a * c; // byte * byte = int (auto promoted)
        System.out.println(result); // 300
    }
}
```

## Type Casting (Manual / Narrowing)
- Big type --> Small type = **manual casting** needed (possible data loss)
- You tell Java: "I know there might be data loss, do it anyway"

```java
class Hello {
    public static void main(String args[]) {

        // Manual casting - big to small
        int num = 10;
        // byte b = num;       // ERROR! int is bigger than byte
        byte b = (byte) num;   // OK! casting with (byte)
        System.out.println(b); // 10

        // What happens when value is OUT OF RANGE?
        int big = 257;
        byte small = (byte) big;
        System.out.println(small); // 1 (because 257 % 256 = 1)

        // Float to int - decimal part is LOST
        float f = 9.8f;
        int x = (int) f;
        System.out.println(x);  // 9 (decimal removed, NOT rounded)

        // double to int
        double d = 99.99;
        int y = (int) d;
        System.out.println(y);  // 99
    }
}
```

## Quick Chart
```
Automatic (Widening):  byte --> short --> int --> long --> float --> double
Manual (Narrowing):    double --> float --> long --> int --> short --> byte
```
