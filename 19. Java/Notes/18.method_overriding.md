# 18. Method Overriding

> ref: `Code/22.MethodOverriding/`

- Child class **re-defines** a method that exists in parent class
- Same method name, same parameters, same return type
- This is **Runtime Polymorphism**

## Overloading vs Overriding

| Feature          | Overloading              | Overriding                |
|------------------|--------------------------|---------------------------|
| Where?           | Same class               | Parent-Child classes      |
| Parameters       | Must be different        | Must be same              |
| Return type      | Can be different         | Must be same              |
| When decided?    | Compile time             | Runtime                   |
| Keyword          | -                        | `@Override` (optional)    |

## Example

```java
class Animal {
    void sound() {
        System.out.println("Animal makes sound");
    }

    void info() {
        System.out.println("I am an animal");
    }
}

class Dog extends Animal {

    @Override // tells compiler: "I am overriding parent's method"
    void sound() {
        System.out.println("Dog barks! Woof Woof!");
    }

    // info() is NOT overridden - Dog inherits it as-is
}

class Cat extends Animal {

    @Override
    void sound() {
        System.out.println("Cat meows! Meow!");
    }
}

class Hello {
    public static void main(String args[]) {

        Dog d = new Dog();
        d.sound(); // Dog barks! Woof Woof! (overridden)
        d.info();  // I am an animal (inherited, not overridden)

        Cat c = new Cat();
        c.sound(); // Cat meows! Meow!

        // Runtime Polymorphism - parent ref, child object
        Animal a = new Dog();
        a.sound(); // Dog barks! Woof Woof! (Dog's method called, not Animal's)
    }
}
```

## Rules of Overriding
```java
class Parent {
    // 1. Cannot override static methods (they are hidden, not overridden)
    static void staticMethod() { }

    // 2. Cannot override final methods
    final void finalMethod() { }

    // 3. Cannot override private methods (not visible to child)
    private void privateMethod() { }

    // 4. CAN override protected and public methods
    protected void protectedMethod() { }
    public void publicMethod() { }
}

class Child extends Parent {
    // static void staticMethod() { }  // This is METHOD HIDING, not overriding
    // void finalMethod() { }          // ERROR! Cannot override final
    // void privateMethod() { }        // This is a NEW method, not override

    @Override
    protected void protectedMethod() { } // OK
    @Override
    public void publicMethod() { }       // OK
}
```

## Interview Points
- `@Override` annotation is optional but recommended (catches typos)
- Access modifier in child can be SAME or WIDER (not narrower)
  - Parent: `protected` --> Child can be: `protected` or `public` (not `private`)
- `static` methods are hidden, not overridden
- `final` methods cannot be overridden
- `private` methods cannot be overridden (not inherited)
