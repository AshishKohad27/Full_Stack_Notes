# Exception Handling

- Exception = error that happens **during runtime** (not compile time)
- Without handling, your program **crashes**
- With handling, your program **continues running**

## 1. try-catch - Basic

```java
class Hello {
    public static void main(String args[]) {

        // WITHOUT try-catch - program CRASHES
        // int a = 10;
        // int b = 0;
        // int result = a / b; // ArithmeticException: / by zero --> CRASH!

        // WITH try-catch - program CONTINUES
        try
        {
            int a = 10;
            int b = 0;
            int result = a / b; // error happens here
            System.out.println(result); // this line will NOT run
        }
        catch (ArithmeticException e)
        {
            System.out.println("Cannot divide by zero!");
            System.out.println("Error: " + e.getMessage()); // Error: / by zero
        }

        System.out.println("Program continues running!"); // this WILL run
    }
}
```

## 2. Multiple catch blocks

```java
class Hello {
    public static void main(String args[]) {

        try
        {
            int arr[] = {1, 2, 3};

            // This will throw ArrayIndexOutOfBoundsException
            System.out.println(arr[5]);

            // This would throw ArithmeticException
            int result = 10 / 0;
        }
        catch (ArrayIndexOutOfBoundsException e)
        {
            System.out.println("Array index is wrong! " + e.getMessage());
        }
        catch (ArithmeticException e)
        {
            System.out.println("Math error! " + e.getMessage());
        }
        catch (Exception e) // catches ANY exception (parent of all exceptions)
        {
            System.out.println("Something went wrong: " + e.getMessage());
        }
    }
}
```

## 3. finally - ALWAYS runs (even if error happens)

```java
class Hello {
    public static void main(String args[]) {

        try
        {
            int result = 10 / 0;
        }
        catch (Exception e)
        {
            System.out.println("Error caught!");
        }
        finally
        {
            // This block ALWAYS runs - error or no error
            // Used for cleanup: closing files, database connections etc.
            System.out.println("Finally block - I ALWAYS run!");
        }
    }
}
// Output:
// Error caught!
// Finally block - I ALWAYS run!
```

## 4. throw - Manually throw an exception

```java
class Hello {
    public static void main(String args[]) {

        int age = 15;

        if (age < 18)
        {
            // YOU throw the exception manually
            throw new ArithmeticException("Age must be 18 or above!");
        }

        System.out.println("Welcome!"); // won't run if age < 18
    }
}
// Output: Exception in thread "main" java.lang.ArithmeticException: Age must be 18 or above!
```

## 5. throws - Declare that method MIGHT throw exception

```java
class Hello {

    // "throws" tells caller: "this method might throw an exception, handle it!"
    static int divide(int a, int b) throws ArithmeticException
    {
        return a / b;
    }

    public static void main(String args[]) {

        try
        {
            int result = divide(10, 0);
            System.out.println(result);
        }
        catch (ArithmeticException e)
        {
            System.out.println("Cannot divide by zero!");
        }
    }
}
```

## 6. Custom Exception - Create your own exception

```java
// Step 1: Create your own exception class
class AgeException extends Exception
{
    AgeException(String message)
    {
        super(message); // pass message to parent Exception class
    }
}

class Hello {

    static void checkAge(int age) throws AgeException
    {
        if (age < 18)
        {
            throw new AgeException("You are " + age + " - Must be 18+");
        }
        System.out.println("Welcome! Age is valid.");
    }

    public static void main(String args[]) {

        try
        {
            checkAge(15); // throws AgeException
        }
        catch (AgeException e)
        {
            System.out.println("Error: " + e.getMessage());
            // Error: You are 15 - Must be 18+
        }
    }
}
```

## Common Exceptions to Know
| Exception                        | When it happens                    |
|----------------------------------|------------------------------------|
| `ArithmeticException`            | Divide by zero                     |
| `ArrayIndexOutOfBoundsException` | Wrong array index                  |
| `NullPointerException`           | Using null object                  |
| `NumberFormatException`          | Converting wrong string to number  |
| `StringIndexOutOfBoundsException`| Wrong string index                 |

## Key Points
- `try` = code that might cause error
- `catch` = what to do if error happens
- `finally` = always runs (cleanup code)
- `throw` = YOU throw an exception manually
- `throws` = declare that method might throw exception
- `throw` vs `throws`: throw is INSIDE method, throws is ON method signature
