# 19. Packages & Imports

> ref: `Code/24.packages/`

- Package = **folder** to organize classes
- Prevents name conflicts (two classes can have same name in different packages)
- Use `import` to use classes from other packages

## 1. Creating a Package

```java
// File: tools/Calculator.java
package tools; // this class belongs to "tools" package

public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }

    public int sub(int a, int b) {
        return a - b;
    }
}
```

```java
// File: Hello.java
import tools.Calculator; // import Calculator from tools package

class Hello {
    public static void main(String args[]) {

        Calculator calc = new Calculator();
        System.out.println(calc.add(5, 3));  // 8
        System.out.println(calc.sub(10, 4)); // 6
    }
}
```

## 2. Import Styles

```java
import java.util.Scanner;     // import ONE specific class
import java.util.*;            // import ALL classes from java.util package

// Without import - use full path
java.util.Scanner sc = new java.util.Scanner(System.in);
```

## 3. Common Built-in Packages

| Package          | Contains                              | Example Classes          |
|------------------|---------------------------------------|--------------------------|
| `java.lang`      | Core classes (auto-imported)          | String, Math, System     |
| `java.util`      | Collections, Scanner, Date           | ArrayList, HashMap       |
| `java.io`        | Input/Output, File handling           | File, BufferedReader     |
| `java.math`      | Math operations                       | BigInteger, BigDecimal   |

## Compile & Run with Packages
```
Folder Structure:
project/
├── Hello.java
└── tools/
    └── Calculator.java

Compile:  javac tools/Calculator.java
          javac Hello.java
Run:      java Hello
```
