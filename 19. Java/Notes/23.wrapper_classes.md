# 23. Wrapper Classes & Autoboxing

- Wrapper class = **object version** of primitive types
- Needed because Collections (ArrayList, HashMap) work with **objects only**, not primitives

## Primitive vs Wrapper

| Primitive | Wrapper     |
|-----------|-------------|
| `int`     | `Integer`   |
| `double`  | `Double`    |
| `float`   | `Float`     |
| `long`    | `Long`      |
| `char`    | `Character` |
| `boolean` | `Boolean`   |
| `byte`    | `Byte`      |
| `short`   | `Short`     |

## 1. Autoboxing & Unboxing

```java
class Hello {
    public static void main(String args[]) {

        // AUTOBOXING: primitive --> Wrapper (automatic)
        int num = 10;
        Integer obj = num;  // int automatically wrapped to Integer
        System.out.println(obj); // 10

        // UNBOXING: Wrapper --> primitive (automatic)
        Integer obj2 = 20;
        int num2 = obj2;    // Integer automatically unwrapped to int
        System.out.println(num2); // 20

        // Manual way (old style)
        Integer obj3 = Integer.valueOf(30);    // boxing
        int num3 = obj3.intValue();             // unboxing
    }
}
```

## 2. Useful Wrapper Methods

```java
class Hello {
    public static void main(String args[]) {

        // String to int
        String s1 = "123";
        int num = Integer.parseInt(s1);
        System.out.println(num + 1); // 124

        // String to double
        String s2 = "3.14";
        double d = Double.parseDouble(s2);
        System.out.println(d); // 3.14

        // int to String
        int x = 100;
        String str = Integer.toString(x);
        // or: String str = String.valueOf(x);
        // or: String str = x + "";
        System.out.println(str); // "100"

        // Min/Max values
        System.out.println(Integer.MAX_VALUE); // 2147483647
        System.out.println(Integer.MIN_VALUE); // -2147483648

        // Compare
        System.out.println(Integer.compare(10, 20)); // -1 (10 < 20)
        System.out.println(Integer.compare(20, 20)); //  0 (equal)
        System.out.println(Integer.compare(30, 20)); //  1 (30 > 20)
    }
}
```

## 3. Integer Cache -- INTERVIEW TRICK!

```java
class Hello {
    public static void main(String args[]) {

        Integer a = 127;
        Integer b = 127;
        System.out.println(a == b); // true (cached! -128 to 127 are cached)

        Integer c = 128;
        Integer d = 128;
        System.out.println(c == d); // false!! (outside cache range)

        // ALWAYS use .equals() for comparing Wrapper objects
        System.out.println(c.equals(d)); // true
    }
}
```

## Interview Points
- Autoboxing: `int` --> `Integer` (automatic)
- Unboxing: `Integer` --> `int` (automatic)
- Integer cache: -128 to 127 are cached (same reference)
- Use `.equals()` not `==` for comparing wrapper objects
- `Integer.parseInt("123")` converts String to int
- `NullPointerException` if you unbox `null`: `Integer x = null; int y = x; // CRASH!`
