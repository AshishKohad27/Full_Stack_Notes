# Abstract Class

- Abstract class is a class that **cannot create objects** directly
- It can have **abstract methods** (methods without body) and **normal methods** (with body)
- If a class has even ONE abstract method, the class MUST be abstract
- Child class **must** implement all abstract methods

## Why use Abstract Class?
- When you want to give a **blueprint** - "every child MUST have these methods"
- Like a form template - the structure is fixed, but you fill in the details

## Example 1 - Basic Abstract Class

```java
// You CANNOT do: Shape s = new Shape(); // ERROR! Cannot create object of abstract class
abstract class Shape
{
    // Abstract method - NO body, just declaration
    // Every child class MUST write their own version
    abstract void draw();

    // Normal method - has body, child gets it for free
    void info()
    {
        System.out.println("I am a shape");
    }
}

// Circle MUST implement draw() because Shape says so
class Circle extends Shape
{
    void draw()
    {
        System.out.println("Drawing Circle ⭕");
    }
}

// Rectangle MUST implement draw() because Shape says so
class Rectangle extends Shape
{
    void draw()
    {
        System.out.println("Drawing Rectangle 🟥");
    }
}

class Hello {
    public static void main(String args[]) {

        // Shape s = new Shape(); // ERROR! Cannot create object of abstract class

        Circle c = new Circle();
        c.draw();  // Drawing Circle ⭕
        c.info();  // I am a shape (inherited from Shape)

        Rectangle r = new Rectangle();
        r.draw();  // Drawing Rectangle 🟥
        r.info();  // I am a shape (inherited from Shape)
    }
}
```

## Example 2 - Real World Example

```java
abstract class Car
{
    String brand;
    int speed;

    // Constructor - yes, abstract class CAN have constructor
    Car(String brand, int speed)
    {
        this.brand = brand;
        this.speed = speed;
    }

    // Abstract method - each car starts differently
    abstract void start();

    // Normal method - all cars stop the same way
    void stop()
    {
        System.out.println(brand + " stopped!");
    }

    void showInfo()
    {
        System.out.println("Brand: " + brand + ", Speed: " + speed + " km/h");
    }
}

class Tesla extends Car
{
    Tesla(int speed)
    {
        super("Tesla", speed); // calling parent constructor
    }

    void start()
    {
        System.out.println("Tesla starts silently... (Electric)");
    }
}

class BMW extends Car
{
    BMW(int speed)
    {
        super("BMW", speed);
    }

    void start()
    {
        System.out.println("BMW starts with ROAR... (Petrol)");
    }
}

class Hello {
    public static void main(String args[]) {

        Tesla t = new Tesla(200);
        t.showInfo();  // Brand: Tesla, Speed: 200 km/h
        t.start();     // Tesla starts silently... (Electric)
        t.stop();      // Tesla stopped!

        System.out.println("---");

        BMW b = new BMW(250);
        b.showInfo();  // Brand: BMW, Speed: 250 km/h
        b.start();     // BMW starts with ROAR... (Petrol)
        b.stop();      // BMW stopped!
    }
}
```

## Key Points
- `abstract class` = cannot create object, can have abstract + normal methods
- `abstract method` = no body, child MUST implement it
- Abstract class CAN have constructor, variables, normal methods
- If you don't implement abstract method in child, child also becomes abstract
