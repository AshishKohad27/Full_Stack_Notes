# 17. Constructor - Deep Dive

## 1. Default vs Parameterized vs Copy Constructor

```java
class Employee {
    String name;
    double salary;

    // Default Constructor
    Employee() {
        name = "Unknown";
        salary = 0;
    }

    // Parameterized Constructor
    Employee(String name, double salary) {
        this.name = name;
        this.salary = salary;
    }

    // Copy Constructor - creates copy of another object
    Employee(Employee other) {
        this.name = other.name;
        this.salary = other.salary;
    }

    void display() {
        System.out.println(name + " - Rs." + salary);
    }
}

class Hello {
    public static void main(String args[]) {

        Employee e1 = new Employee();                  // Default
        Employee e2 = new Employee("Ashish", 50000);   // Parameterized
        Employee e3 = new Employee(e2);                 // Copy of e2

        e1.display(); // Unknown - Rs.0.0
        e2.display(); // Ashish - Rs.50000.0
        e3.display(); // Ashish - Rs.50000.0 (copy of e2)
    }
}
```

## 2. Constructor Chaining (this + super)

```java
class Animal {
    String type;

    Animal(String type) {
        this.type = type;
        System.out.println("Animal: " + type);
    }
}

class Dog extends Animal {
    String breed;

    Dog() {
        this("Unknown"); // calls Dog(String breed) below
        System.out.println("Default Dog constructor");
    }

    Dog(String breed) {
        super("Dog"); // calls Animal(String type) - MUST be first line
        this.breed = breed;
        System.out.println("Breed: " + breed);
    }
}

class Hello {
    public static void main(String args[]) {

        Dog d = new Dog();
        // Output:
        // Animal: Dog         (super called first)
        // Breed: Unknown      (this() constructor body)
        // Default Dog constructor
    }
}
```

## Interview Points
- If you define ANY constructor, Java won't give default constructor
- `this()` and `super()` MUST be first statement (can't use both in same constructor)
- Constructor is NOT inherited
- Constructor can be `private` (used in Singleton pattern)
- Constructor CANNOT be `abstract`, `static`, or `final`
