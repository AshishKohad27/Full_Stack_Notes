# Interface

- Interface is like a **100% abstract class** - ALL methods are abstract (no body)
- A class can implement **multiple interfaces** (but can extend only ONE class)
- Interface is like a **contract** - "if you sign it, you MUST do everything in it"
- Variables in interface are `public static final` by default (constants)
- Methods in interface are `public abstract` by default

## Interface vs Abstract Class
| Feature         | Interface                    | Abstract Class              |
|-----------------|------------------------------|-----------------------------|
| Methods         | All abstract (before Java 8) | Can have abstract + normal  |
| Variables       | Only constants               | Can have any variables      |
| Multiple        | Yes (implement many)         | No (extend only one)        |
| Constructor     | No                           | Yes                         |
| Keyword         | `implements`                 | `extends`                   |

## Example 1 - Basic Interface

```java
// Interface - just method names, no body
interface Animal
{
    void sound();   // public abstract by default
    void eat();     // public abstract by default
}

// Dog MUST implement ALL methods of Animal
class Dog implements Animal
{
    public void sound()
    {
        System.out.println("Dog: Bark Bark!");
    }

    public void eat()
    {
        System.out.println("Dog: Eats bones");
    }
}

class Cat implements Animal
{
    public void sound()
    {
        System.out.println("Cat: Meow Meow!");
    }

    public void eat()
    {
        System.out.println("Cat: Eats fish");
    }
}

class Hello {
    public static void main(String args[]) {

        Dog d = new Dog();
        d.sound();  // Dog: Bark Bark!
        d.eat();    // Dog: Eats bones

        Cat c = new Cat();
        c.sound();  // Cat: Meow Meow!
        c.eat();    // Cat: Eats fish
    }
}
```

## Example 2 - Multiple Interfaces (Main Advantage!)

```java
// A class can extend only ONE class
// But a class can implement MULTIPLE interfaces

interface Playable
{
    void play();
}

interface Recordable
{
    void record();
}

interface Callable
{
    void call();
}

// Phone implements ALL three interfaces
class Phone implements Playable, Recordable, Callable
{
    public void play()
    {
        System.out.println("Playing music...");
    }

    public void record()
    {
        System.out.println("Recording video...");
    }

    public void call()
    {
        System.out.println("Calling someone...");
    }
}

class Hello {
    public static void main(String args[]) {

        Phone p = new Phone();
        p.play();    // Playing music...
        p.record();  // Recording video...
        p.call();    // Calling someone...
    }
}
```

## Example 3 - Interface with Reference Type

```java
interface Animal
{
    void sound();
}

class Dog implements Animal
{
    public void sound()
    {
        System.out.println("Bark!");
    }
}

class Cat implements Animal
{
    public void sound()
    {
        System.out.println("Meow!");
    }
}

class Hello {
    public static void main(String args[]) {

        // You can use Interface as reference type
        Animal a1 = new Dog(); // Interface reference, Dog object
        Animal a2 = new Cat(); // Interface reference, Cat object

        a1.sound(); // Bark!
        a2.sound(); // Meow!

        // This is POLYMORPHISM - same type (Animal), different behavior
    }
}
```

## Key Points
- Interface = contract, all methods must be implemented
- Use `implements` keyword (not extends)
- A class can implement **multiple** interfaces: `class A implements X, Y, Z`
- Interface **cannot** have constructor
- All variables are `public static final` (constants)
- All methods are `public abstract` (before Java 8)
