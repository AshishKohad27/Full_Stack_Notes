# 12. OOP - Object Oriented Programming

> ref: `Code/10.OOP/`

- OOP = organizing code using **Classes** and **Objects**
- 4 Pillars of OOP: **Encapsulation, Inheritance, Polymorphism, Abstraction**

## What is Class and Object?

```
Class  = Blueprint / Template (like a house plan)
Object = Actual thing created from blueprint (like the actual house)
```

## 1. Class & Object - Basic

```java
// Class = Blueprint
class Car {
    // Properties (what it HAS)
    String brand;
    String color;
    int speed;

    // Methods (what it DOES)
    void drive() {
        System.out.println(brand + " is driving at " + speed + " km/h");
    }

    void stop() {
        System.out.println(brand + " stopped!");
    }
}

class Hello {
    public static void main(String args[]) {

        // Object = actual car created from blueprint
        Car car1 = new Car();
        car1.brand = "Tesla";
        car1.color = "White";
        car1.speed = 200;

        Car car2 = new Car();
        car2.brand = "BMW";
        car2.color = "Black";
        car2.speed = 250;

        car1.drive(); // Tesla is driving at 200 km/h
        car2.drive(); // BMW is driving at 250 km/h
        car2.stop();  // BMW stopped!

        // car1 and car2 are DIFFERENT objects of SAME class
    }
}
```

## 2. Constructor (Initialize object at creation)

```java
class Student {
    String name;
    int marks;

    // Constructor - same name as class, no return type
    Student(String name, int marks) {
        this.name = name;   // this.name = object's variable
        this.marks = marks;
    }

    void display() {
        System.out.println(name + " : " + marks);
    }
}

class Hello {
    public static void main(String args[]) {

        // Values assigned AT creation time
        Student s1 = new Student("Ashish", 90);
        Student s2 = new Student("Rahul", 85);

        s1.display(); // Ashish : 90
        s2.display(); // Rahul : 85
    }
}
```

## 3. The 4 Pillars Quick Overview

```
1. ENCAPSULATION - Wrapping data + methods together, hiding internal details
   Example: private variables + public getters/setters

2. INHERITANCE - Child class gets properties of parent class
   Example: class Dog extends Animal

3. POLYMORPHISM - Same method behaves differently
   Example: Animal a = new Dog(); a.sound(); // calls Dog's sound

4. ABSTRACTION - Hiding complex implementation, showing only what's needed
   Example: You press power button on TV, you don't know the circuit inside
```

## Interview Favorite: Class vs Object

| Class                           | Object                          |
|---------------------------------|---------------------------------|
| Blueprint / Template            | Actual instance                 |
| No memory allocated             | Memory allocated (in heap)      |
| Created once                    | Can create many from one class  |
| Logical entity                  | Physical entity                 |
| `class Car { }`                 | `Car c = new Car();`            |
