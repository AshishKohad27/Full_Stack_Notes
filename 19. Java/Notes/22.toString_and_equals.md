# 22. toString() and equals() Methods

- Every class extends `Object` class
- `Object` has methods like `toString()`, `equals()`, `hashCode()`
- You should **override** them for meaningful output

## 1. toString() - String representation of object

```java
class Student {
    String name;
    int marks;

    Student(String name, int marks) {
        this.name = name;
        this.marks = marks;
    }

    // WITHOUT toString() override:
    // System.out.println(s1) prints something like: Student@15db9742 (useless!)

    // WITH toString() override:
    @Override
    public String toString() {
        return "Student{name='" + name + "', marks=" + marks + "}";
    }
}

class Hello {
    public static void main(String args[]) {

        Student s1 = new Student("Ashish", 90);
        System.out.println(s1); // Student{name='Ashish', marks=90}
        // println automatically calls toString()
    }
}
```

## 2. equals() - Compare two objects by VALUE

```java
class Student {
    String name;
    int marks;

    Student(String name, int marks) {
        this.name = name;
        this.marks = marks;
    }

    // WITHOUT equals() override:
    // s1 == s2 compares REFERENCES (memory address), not values
    // s1.equals(s2) also compares references by default

    // WITH equals() override:
    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true;               // same object
        if (obj == null) return false;               // null check
        if (getClass() != obj.getClass()) return false; // different class
        Student other = (Student) obj;               // cast to Student
        return this.name.equals(other.name) && this.marks == other.marks;
    }
}

class Hello {
    public static void main(String args[]) {

        Student s1 = new Student("Ashish", 90);
        Student s2 = new Student("Ashish", 90);
        Student s3 = new Student("Rahul", 85);

        // Without override: s1.equals(s2) would be FALSE (different objects)
        // With override: compares by name and marks
        System.out.println(s1.equals(s2)); // true (same values)
        System.out.println(s1.equals(s3)); // false (different values)
        System.out.println(s1 == s2);      // false (still different references!)
    }
}
```

## Interview Points
- `==` compares **reference** (memory address)
- `.equals()` compares **value** (if overridden properly)
- Always override `equals()` when you need value-based comparison
- If you override `equals()`, also override `hashCode()` (HashMap uses both)
- `toString()` is called automatically by `println()`, string concatenation etc.
