# 39. Interview Tricky Concepts - Quick Revision

## 1. Pass by Value (Java is ALWAYS pass by value)

```java
class Hello {

    static void change(int x) {
        x = 100; // changes LOCAL copy, not original
    }

    static void changeArray(int[] arr) {
        arr[0] = 999; // changes ACTUAL array (reference is passed by value)
    }

    public static void main(String args[]) {

        // Primitive - pass by value (copy)
        int num = 10;
        change(num);
        System.out.println(num); // 10 (NOT 100! original unchanged)

        // Object/Array - reference is passed by value
        int[] arr = {1, 2, 3};
        changeArray(arr);
        System.out.println(arr[0]); // 999 (changed! because reference points to same array)
    }
}
```
**Interview Answer:** Java is always pass by value. For primitives, the value is copied. For objects, the reference is copied (not the object), so you can modify the object's content but not reassign the reference.

## 2. String Pool & Immutability

```java
class Hello {
    public static void main(String args[]) {

        String s1 = "Hello"; // goes to String Pool
        String s2 = "Hello"; // reuses same pool object
        String s3 = new String("Hello"); // creates NEW object in heap

        System.out.println(s1 == s2);      // true (same pool reference)
        System.out.println(s1 == s3);      // false (different objects!)
        System.out.println(s1.equals(s3)); // true (same value)

        // WHY is String immutable?
        // 1. String Pool works because strings don't change
        // 2. Thread safety - multiple threads can share same string
        // 3. Security - class names, file paths are strings
        // 4. hashCode caching - computed once, never changes
    }
}
```

## 3. == vs .equals() vs hashCode()

```java
class Hello {
    public static void main(String args[]) {

        // ==          : compares REFERENCE (memory address)
        // .equals()   : compares VALUE (if overridden)
        // hashCode()  : returns integer hash of object

        String a = new String("Hi");
        String b = new String("Hi");

        System.out.println(a == b);       // false (different objects)
        System.out.println(a.equals(b));  // true (same value)
        System.out.println(a.hashCode() == b.hashCode()); // true (same value = same hash)

        // RULE: If equals() is true, hashCode() MUST be same
        // But same hashCode doesn't guarantee equals() is true (hash collision)
    }
}
```

## 4. Static Binding vs Dynamic Binding

```java
class Parent {
    static void greet() { System.out.println("Parent static"); }
    void show() { System.out.println("Parent show"); }
}

class Child extends Parent {
    static void greet() { System.out.println("Child static"); } // method HIDING
    void show() { System.out.println("Child show"); }            // method OVERRIDING
}

class Hello {
    public static void main(String args[]) {

        Parent p = new Child();

        p.greet(); // "Parent static" -- static = compile-time binding (based on reference type)
        p.show();  // "Child show"    -- non-static = runtime binding (based on actual object)
    }
}
```

## 5. Singleton Pattern (Only ONE object allowed)

```java
class Database {
    private static Database instance; // the single instance

    private Database() { } // private constructor - no one can create object from outside

    public static Database getInstance() {
        if (instance == null) {
            instance = new Database(); // create only if not exists
        }
        return instance;
    }

    public void connect() {
        System.out.println("Connected to DB");
    }
}

class Hello {
    public static void main(String args[]) {

        // Database db = new Database(); // ERROR! Constructor is private

        Database db1 = Database.getInstance();
        Database db2 = Database.getInstance();

        System.out.println(db1 == db2); // true (same object!)
        db1.connect(); // Connected to DB
    }
}
```

## 6. Garbage Collection

```java
class Hello {
    public static void main(String args[]) {

        // Java automatically deletes objects that are no longer referenced
        String s = new String("Hello");
        s = null; // now the "Hello" object has no reference
        // Garbage Collector will eventually clean it up

        // You can REQUEST gc (but can't force it)
        System.gc(); // requests garbage collection

        // finalize() method is called before object is garbage collected
        // (deprecated in Java 9+)
    }
}
```

## 7. Varargs (Variable Arguments)

```java
class Hello {

    // ... means "any number of arguments"
    static int sum(int... nums) {
        int total = 0;
        for (int n : nums) {
            total += n;
        }
        return total;
    }

    public static void main(String args[]) {

        System.out.println(sum(1, 2));           // 3
        System.out.println(sum(1, 2, 3, 4, 5));  // 15
        System.out.println(sum());                // 0

        // Varargs must be the LAST parameter
        // static void test(int... nums, String name) // ERROR!
        // static void test(String name, int... nums) // OK!
    }
}
```

## 8. Anonymous Class vs Lambda

```java
interface Greeting {
    void greet();
}

class Hello {
    public static void main(String args[]) {

        // Anonymous class (verbose)
        Greeting g1 = new Greeting() {
            public void greet() {
                System.out.println("Hello from anonymous class!");
            }
        };

        // Lambda (clean) - same thing, shorter syntax
        Greeting g2 = () -> System.out.println("Hello from lambda!");

        g1.greet();
        g2.greet();
    }
}
```

## 9. Memory: Stack vs Heap

```
STACK:                          HEAP:
- Method calls                  - Objects (new keyword)
- Local variables               - Instance variables
- Primitive values              - Arrays
- Reference variables           - Strings
- Fast, small                   - Slow, large
- Auto cleaned (method ends)    - Garbage collected

Example:
  int x = 10;           // x is on Stack, 10 is on Stack
  String s = new String("Hi"); // s (reference) on Stack, "Hi" object on Heap
  int[] arr = {1,2,3};  // arr (reference) on Stack, array object on Heap
```

## 10. Quick Fire Interview Answers

| Question | Answer |
|----------|--------|
| Is Java 100% OOP? | No. It has primitives (int, char) which are not objects |
| Can main() be overloaded? | Yes. But JVM calls only `public static void main(String[])` |
| Can constructor be private? | Yes. Used in Singleton pattern |
| What is Object class? | Parent of all classes. Every class extends Object |
| Difference: throw vs throws? | throw = inside method (throws exception), throws = on method signature (declares) |
| What is autoboxing? | int --> Integer (automatic conversion) |
| What is marker interface? | Interface with no methods (Serializable, Cloneable) |
| Can interface have constructor? | No |
| Can abstract class be final? | No (final = can't extend, abstract = must extend -- contradiction!) |
| What is functional interface? | Interface with exactly ONE abstract method (used with lambda) |
