# 14. Inheritance

> ref: `Code/21.inheretance/`

- Inheritance = child class **gets all properties & methods** of parent class
- Use `extends` keyword
- Promotes **code reusability** - don't repeat same code

## 1. Basic Inheritance

```java
// Parent Class (Super Class / Base Class)
class Animal {
    String name;

    void eat() {
        System.out.println(name + " is eating");
    }

    void sleep() {
        System.out.println(name + " is sleeping");
    }
}

// Child Class (Sub Class / Derived Class) - GETS everything from Animal
class Dog extends Animal {

    // Dog has its OWN method too
    void bark() {
        System.out.println(name + " is barking! Woof!");
    }
}

class Cat extends Animal {
    void meow() {
        System.out.println(name + " says Meow!");
    }
}

class Hello {
    public static void main(String args[]) {

        Dog d = new Dog();
        d.name = "Tommy";
        d.eat();    // Tommy is eating (inherited from Animal)
        d.sleep();  // Tommy is sleeping (inherited from Animal)
        d.bark();   // Tommy is barking! Woof! (Dog's own method)

        Cat c = new Cat();
        c.name = "Kitty";
        c.eat();    // Kitty is eating (inherited)
        c.meow();   // Kitty says Meow!
    }
}
```

## 2. Types of Inheritance

```
1. Single:      A --> B              (B extends A)
2. Multilevel:  A --> B --> C        (C extends B, B extends A)
3. Hierarchical: A --> B, A --> C    (B and C both extend A)

NOTE: Java does NOT support Multiple Inheritance with classes
      A --> C <-- B  (C extends A, B)  --> NOT ALLOWED!
      But you can achieve it with INTERFACES
```

## 3. Multilevel Inheritance

```java
class Vehicle {
    void start() {
        System.out.println("Vehicle started");
    }
}

class Car extends Vehicle {
    void drive() {
        System.out.println("Car is driving");
    }
}

class SportsCar extends Car {
    void turbo() {
        System.out.println("Turbo activated!");
    }
}

class Hello {
    public static void main(String args[]) {

        SportsCar sc = new SportsCar();
        sc.start();  // from Vehicle (grandparent)
        sc.drive();  // from Car (parent)
        sc.turbo();  // own method
    }
}
```

## 4. super keyword

```java
class Animal {
    String type = "Animal";

    Animal() {
        System.out.println("Animal constructor called");
    }

    void sound() {
        System.out.println("Some animal sound");
    }
}

class Dog extends Animal {
    String type = "Dog";

    Dog() {
        super(); // calls parent constructor (called automatically even if you don't write it)
        System.out.println("Dog constructor called");
    }

    void sound() {
        super.sound();  // calls PARENT's sound method
        System.out.println("Dog barks!");
    }

    void showType() {
        System.out.println(type);       // Dog (own variable)
        System.out.println(super.type); // Animal (parent's variable)
    }
}

class Hello {
    public static void main(String args[]) {

        Dog d = new Dog();
        // Output:
        // Animal constructor called  (super constructor runs FIRST)
        // Dog constructor called

        d.sound();
        // Some animal sound  (parent's sound via super.sound())
        // Dog barks!

        d.showType();
        // Dog
        // Animal
    }
}
```

## Interview Points
- Every class extends `Object` class by default (the root of all classes)
- `super()` is called automatically in constructor (even if you don't write it)
- Java does NOT support multiple inheritance with classes (diamond problem)
- Constructor is NOT inherited, but parent constructor is CALLED via `super()`
- `private` members are NOT inherited (not accessible in child)
