# 37. Multithreading Basics

- Thread = a **lightweight process** (a separate path of execution)
- Multithreading = running **multiple threads at the same time**
- Example: You can listen to music AND browse internet simultaneously

## 1. Creating Thread - Way 1: Extend Thread class

```java
class MyThread extends Thread {
    public void run() { // this method runs in a separate thread
        for (int i = 1; i <= 5; i++) {
            System.out.println("Thread: " + i);
            try { Thread.sleep(500); } catch (Exception e) { } // wait 500ms
        }
    }
}

class Hello {
    public static void main(String args[]) {

        MyThread t = new MyThread();
        t.start(); // start() creates new thread and calls run()
        // t.run(); // WRONG! This runs in SAME thread (no new thread created)

        for (int i = 1; i <= 5; i++) {
            System.out.println("Main: " + i);
            try { Thread.sleep(500); } catch (Exception e) { }
        }

        // Output (interleaved - both run simultaneously):
        // Main: 1
        // Thread: 1
        // Main: 2
        // Thread: 2
        // ... (order may vary!)
    }
}
```

## 2. Creating Thread - Way 2: Implement Runnable (Preferred)

```java
class MyTask implements Runnable {
    public void run() {
        for (int i = 1; i <= 5; i++) {
            System.out.println(Thread.currentThread().getName() + ": " + i);
        }
    }
}

class Hello {
    public static void main(String args[]) {

        MyTask task = new MyTask();
        Thread t1 = new Thread(task, "Worker-1"); // name the thread
        Thread t2 = new Thread(task, "Worker-2");

        t1.start();
        t2.start();
    }
}
```

## 3. Thread with Lambda (Cleanest Way)

```java
class Hello {
    public static void main(String args[]) {

        Thread t = new Thread(() -> {
            for (int i = 1; i <= 5; i++) {
                System.out.println("Lambda Thread: " + i);
            }
        });
        t.start();
    }
}
```

## 4. Thread Methods

```java
class Hello {
    public static void main(String args[]) throws Exception {

        Thread t = new Thread(() -> {
            System.out.println("Thread running...");
            try { Thread.sleep(2000); } catch (Exception e) { }
            System.out.println("Thread done!");
        });

        t.start();
        System.out.println("Is alive: " + t.isAlive()); // true
        t.join(); // main thread WAITS for t to finish
        System.out.println("Is alive: " + t.isAlive()); // false
        System.out.println("Main continues after thread is done");
    }
}
```

## Thread States (Lifecycle)
```
NEW --> RUNNABLE --> RUNNING --> (BLOCKED/WAITING) --> TERMINATED
  |        |           |              |
 start() scheduler   sleep/wait    completed
         picks it    join/lock
```

## Interview Points
- `start()` creates new thread, `run()` runs in same thread
- Implement `Runnable` is preferred over extending `Thread` (Java allows only single inheritance)
- `Thread.sleep(ms)` pauses current thread
- `join()` makes one thread wait for another to finish
- `synchronized` keyword prevents race conditions (two threads accessing same data)
- `volatile` keyword ensures variable is read from main memory, not cache
