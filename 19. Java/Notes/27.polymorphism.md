# Polymorphism

- Polymorphism = "Many Forms"
- Same method name behaves **differently** depending on the object
- Two Types:
  1. **Compile-time** polymorphism = Method Overloading (already covered in topic 12)
  2. **Runtime** polymorphism = Method Overriding (parent reference, child object)

## Runtime Polymorphism (Most Important!)

```java
class Animal
{
    void sound()
    {
        System.out.println("Animal makes a sound");
    }
}

class Dog extends Animal
{
    void sound()
    {
        System.out.println("Dog: Bark Bark!");
    }
}

class Cat extends Animal
{
    void sound()
    {
        System.out.println("Cat: Meow Meow!");
    }
}

class Cow extends Animal
{
    void sound()
    {
        System.out.println("Cow: Moo Moo!");
    }
}

class Hello {
    public static void main(String args[]) {

        // PARENT reference = CHILD object
        // This is the key to polymorphism
        Animal a1 = new Dog();  // Animal type but Dog object
        Animal a2 = new Cat();  // Animal type but Cat object
        Animal a3 = new Cow();  // Animal type but Cow object

        a1.sound(); // Dog: Bark Bark!    (calls Dog's sound, not Animal's)
        a2.sound(); // Cat: Meow Meow!    (calls Cat's sound, not Animal's)
        a3.sound(); // Cow: Moo Moo!      (calls Cow's sound, not Animal's)

        // Java decides WHICH sound() to call at RUNTIME based on actual object
        // This is why it's called "Runtime Polymorphism"
    }
}
```

## Real World Example - Why Polymorphism is Useful

```java
class Payment
{
    void pay(int amount)
    {
        System.out.println("Paying " + amount);
    }
}

class CreditCard extends Payment
{
    void pay(int amount)
    {
        System.out.println("Paid " + amount + " using Credit Card");
    }
}

class UPI extends Payment
{
    void pay(int amount)
    {
        System.out.println("Paid " + amount + " using UPI");
    }
}

class Cash extends Payment
{
    void pay(int amount)
    {
        System.out.println("Paid " + amount + " using Cash");
    }
}

class Hello {
    public static void main(String args[]) {

        // Without Polymorphism - you need different variables for each type
        // CreditCard c = new CreditCard();
        // UPI u = new UPI();
        // Cash cash = new Cash();

        // With Polymorphism - ONE type handles ALL
        Payment p;

        p = new CreditCard();
        p.pay(500);  // Paid 500 using Credit Card

        p = new UPI();
        p.pay(200);  // Paid 200 using UPI

        p = new Cash();
        p.pay(100);  // Paid 100 using Cash

        // Even better - you can use arrays!
        Payment payments[] = new Payment[3];
        payments[0] = new CreditCard();
        payments[1] = new UPI();
        payments[2] = new Cash();

        // One loop handles ALL payment types
        for (Payment payment : payments)
        {
            payment.pay(1000);
        }
        // Paid 1000 using Credit Card
        // Paid 1000 using UPI
        // Paid 1000 using Cash
    }
}
```

## Quick Summary
- **Compile-time Polymorphism** = Method Overloading (same name, different parameters)
- **Runtime Polymorphism** = Method Overriding (parent ref, child object)
- `Animal a = new Dog();` --> calls Dog's methods, not Animal's
- Useful when you want ONE variable type to handle MANY different objects
