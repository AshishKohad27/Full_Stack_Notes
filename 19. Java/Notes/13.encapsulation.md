# 13. Encapsulation

> ref: `Code/17.Encapsulation/`

- Encapsulation = **wrapping data (variables) and methods together** in a class
- **Hide** internal details using `private`
- **Expose** only what's needed using `public` getters/setters
- Think of it like a **capsule** - medicine is inside, you just swallow it

## Example

```java
class BankAccount {

    // Private - nobody can directly access from outside
    private String owner;
    private double balance;

    // Constructor
    BankAccount(String owner, double balance) {
        this.owner = owner;
        this.balance = balance;
    }

    // GETTER - to READ private variable
    public String getOwner() {
        return owner;
    }

    public double getBalance() {
        return balance;
    }

    // SETTER - to WRITE/MODIFY private variable (with validation!)
    public void setBalance(double amount) {
        if (amount >= 0) {
            balance = amount;
        } else {
            System.out.println("Balance cannot be negative!");
        }
    }

    // Method to deposit
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            System.out.println("Deposited: " + amount + " | Balance: " + balance);
        }
    }

    // Method to withdraw
    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            System.out.println("Withdrawn: " + amount + " | Balance: " + balance);
        } else {
            System.out.println("Insufficient balance!");
        }
    }
}

class Hello {
    public static void main(String args[]) {

        BankAccount acc = new BankAccount("Ashish", 5000);

        // acc.balance = -1000; // ERROR! balance is private (protected from misuse)

        acc.deposit(2000);    // Deposited: 2000 | Balance: 7000
        acc.withdraw(1000);   // Withdrawn: 1000 | Balance: 6000
        acc.withdraw(10000);  // Insufficient balance!

        System.out.println("Owner: " + acc.getOwner());     // Ashish
        System.out.println("Balance: " + acc.getBalance());  // 6000

        acc.setBalance(-500); // Balance cannot be negative!
    }
}
```

## Why Encapsulation?
1. **Data Protection** - nobody can set `balance = -1000` directly
2. **Validation** - setter checks before allowing changes
3. **Flexibility** - you can change internal code without breaking outside code
4. **Control** - you decide what others can see and modify

## Interview Points
- Encapsulation = data hiding + bundling
- Make variables `private`, provide `public` getter/setter
- Getter = read-only access, Setter = write access with validation
- Real world: ATM machine (you use buttons, don't see internal wiring)
