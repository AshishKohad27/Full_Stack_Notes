# 8. CDN (Content Delivery Network)

- CDN = network of servers **spread across the world** to serve content faster
- Stores copies of your **static content** (images, videos, CSS, JS) closer to users
- Like having multiple warehouses in different cities instead of shipping from one central location

## How CDN Works

```
WITHOUT CDN:
  User in India → requests image → Server in USA → sends image
  Distance: 15,000 km → SLOW (500ms)

WITH CDN:
  User in India → requests image → CDN server in India → sends image
  Distance: 100 km → FAST (20ms)
```

```
                     ┌──── CDN Edge (India)
                     │
Origin Server ───────┼──── CDN Edge (USA)
(Your main server)   │
                     ├──── CDN Edge (Europe)
                     │
                     └──── CDN Edge (Japan)

First request: CDN fetches from Origin → caches it
Next requests: CDN serves from its own cache → FAST!
```

## What CDN Stores (Static Content)

```
YES (static - doesn't change per user):
  - Images (profile pics, product photos)
  - Videos (Netflix streams, YouTube)
  - CSS files
  - JavaScript files
  - Fonts
  - PDF documents

NO (dynamic - changes per user):
  - User-specific data (your feed, your cart)
  - API responses (unless cacheable)
  - Real-time data (chat messages, stock prices)
```

## Push CDN vs Pull CDN

### Pull CDN (Most Common)
```
1. User requests image from CDN
2. CDN doesn't have it (first time)
3. CDN "pulls" image from Origin Server
4. CDN caches it
5. Next user asking for same image → CDN serves from cache

You don't need to upload to CDN manually.
CDN pulls content when first requested.
```

### Push CDN
```
1. YOU upload content to CDN manually
2. CDN stores it
3. Users access from CDN

You control exactly what's on CDN.
Good for: content you know will be popular (movie release, new product).
```

## Real World Examples

```
Netflix:
  - Movies/Shows cached on CDN servers worldwide
  - When you watch a movie, it streams from nearest CDN
  - Result: smooth streaming, no buffering

Instagram:
  - All photos/videos served via CDN
  - Your photo is copied to CDN servers globally
  - Friends in any country load it fast
```

## Key Points
- CDN = copies of content placed near users globally
- Makes **static content** load **10-100x faster**
- Reduces load on your **origin server**
- Also provides **DDoS protection** (attack traffic hits CDN, not your server)
- Most websites use CDN (you should too)
- Content expires via TTL, then CDN fetches fresh copy from origin
