# 36. Design: Netflix / YouTube (Video Streaming)

## Step 1: Requirements

```
Functional:
  - Upload videos
  - Stream/watch videos
  - Search videos
  - Recommendations
  - Like, comment, subscribe

Non-Functional:
  - Smooth streaming (no buffering)
  - Support multiple resolutions (240p, 480p, 720p, 1080p, 4K)
  - Global availability
  - Handle millions of concurrent viewers
```

## Step 2: High-Level Design

```
UPLOAD Flow:
  Creator → Upload → API → S3 (raw) → Transcoding Queue → Transcoding Service
                                                              ↓
                                                        S3 (multiple resolutions)
                                                              ↓
                                                           CDN
WATCH Flow:
  Viewer → API → Video Service → CDN → stream video to viewer
```

```
┌──────────┐       ┌───────────┐       ┌──────────────────┐
│ Creator  │──────→│ Upload    │──────→│ Object Storage   │
│ uploads  │       │ Service   │       │ (S3 - raw video) │
└──────────┘       └───────────┘       └────────┬─────────┘
                                                │
                                         ┌──────┴─────────┐
                                         │ Message Queue  │
                                         └──────┬─────────┘
                                                │
                                    ┌───────────┴───────────┐
                                    │ Transcoding Service   │
                                    │ (encode to 240p,480p, │
                                    │  720p, 1080p, 4K)     │
                                    └───────────┬───────────┘
                                                │
                                    ┌───────────┴───────────┐
                                    │ S3 (processed videos) │
                                    └───────────┬───────────┘
                                                │
                                    ┌───────────┴───────────┐

```

## Step 3: Video Upload & Processing

```
1. Creator uploads 1GB raw video
2. Video stored in S3 (original)
3. Message sent to Transcoding Queue
4. Transcoding Service picks up:
   - Encode to multiple resolutions:
     240p  → for slow internet (150 KB/s)
     480p  → for mobile data
     720p  → standard HD
     1080p → full HD
     4K    → ultra HD
   - Generate thumbnails
   - Extract subtitles
   - Create preview clips
5. Processed files stored in S3
6. CDN pulls and caches them globally
7. Video metadata saved to DB (title, description, resolution URLs)

Processing takes TIME (5-30 min for a 10-min video)
→ Show "Processing..." to creator, notify when done
```

## Step 4: Adaptive Bitrate Streaming

```
Problem: User's internet speed changes during video playback.
  Started on WiFi (fast) → moved to mobile data (slow)

Solution: Adaptive Bitrate Streaming (ABR)

Video is split into small CHUNKS (2-10 seconds each):
  Chunk 1: [240p] [480p] [720p] [1080p]
  Chunk 2: [240p] [480p] [720p] [1080p]
  Chunk 3: [240p] [480p] [720p] [1080p]

Player monitors internet speed:
  Fast internet → download 1080p chunks
  Speed drops → switch to 480p chunks seamlessly
  Speed recovers → switch back to 1080p

User sees: Video quality auto-adjusts. No buffering!

Protocols: HLS (Apple) or DASH (standard)
```

## Step 5: CDN is Critical

```
Without CDN:
  Server in USA → Viewer in India → 15,000 km → BUFFERING!

With CDN:
  CDN server in India → Viewer in India → 100 km → SMOOTH!

Netflix has CDN servers (Open Connect) in 1000+ locations worldwide.
Popular content pre-cached everywhere.

When new show launches (Stranger Things):
  → Pre-loaded on ALL CDN servers globally BEFORE release
  → Millions of viewers hit nearest CDN, not origin server
```

## Step 6: Recommendation System (Brief)

```
"How does Netflix know what to suggest?"

Collaborative Filtering:
  "Users who watched Show A also watched Show B"
  If Ashish watched A, B, C and Rahul watched A, B
  → Recommend C to Rahul

Content-Based:
  "You watched action movies → here are more action movies"
  Based on: genre, actors, director, tags

Netflix uses Machine Learning combining both approaches.
```

## Key Design Decisions
```
1. S3 for video storage (cheap, durable, scalable)
2. Transcoding as async background job (message queue)
3. CDN for global streaming performance (THE most important)
4. Adaptive bitrate streaming (auto quality adjustment)
5. Video split into chunks (not one big file)
6. Separate metadata DB from video storage
```
