# 2. Client & Server

- **Client** = the device/browser that REQUESTS data (your phone, laptop)
- **Server** = the computer that PROCESSES requests and SENDS back data

## How It Works

```
CLIENT (Browser/App)          SERVER (Backend)
       |                           |
       |--- "Show me homepage" --->|
       |                           | (processes request)
       |<-- "Here's the HTML" -----|
       |                           |
```

## Example - What Happens When You Open google.com?

```
Step 1: You type google.com in browser (CLIENT)
Step 2: Browser sends HTTP request to Google's SERVER
Step 3: Server processes the request
Step 4: Server sends back HTML, CSS, JS (the webpage)
Step 5: Browser renders (displays) the page
```

## HTTP Methods (How Client Talks to Server)

| Method   | What It Does              | Example                      |
|----------|---------------------------|------------------------------|
| `GET`    | Read/Fetch data           | Get list of products         |
| `POST`   | Create new data           | Create new user account      |
| `PUT`    | Update entire data        | Update user profile          |
| `PATCH`  | Update part of data       | Change only email            |
| `DELETE` | Delete data               | Delete a post                |

## Real Example

```
Instagram App (Client) --> Instagram Server

GET  /feed              --> Server returns your feed posts
POST /post              --> Server creates your new post
PUT  /profile           --> Server updates your profile
DELETE /post/123        --> Server deletes post with id 123
```

## API (Application Programming Interface)

```
API = the RULES/CONTRACT between client and server
    = "If you send me THIS request, I'll give you THIS response"

Example API:
  Request:  GET /users/123
  Response: { "name": "Ashish", "age": 22 }

  Request:  POST /users  { "name": "Rahul", "age": 25 }
  Response: { "message": "User created!", "id": 456 }
```

## REST API vs GraphQL

| Feature    | REST API                        | GraphQL                        |
|------------|----------------------------------|--------------------------------|
| Endpoints  | Multiple (/users, /posts)       | Single (/graphql)              |
| Data       | Server decides what to send     | Client decides what it needs   |
| Over-fetch | Can return extra data           | Returns exactly what you ask   |
| Example    | GET /user/1 returns ALL fields  | query { user(id:1) { name } }  |

## Key Points
- Client sends **request**, Server sends **response**
- Communication happens over **HTTP/HTTPS** protocol
- API defines the contract between client and server
- Server can serve **multiple clients** at the same time
