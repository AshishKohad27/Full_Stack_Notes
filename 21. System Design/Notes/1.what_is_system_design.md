# 1. What is System Design?

- System Design = **planning HOW to build a large-scale application**
- It's like **architecture of a building** - before you build, you plan the structure
- Focuses on: how components talk to each other, where data is stored, how to handle millions of users

## Why Learn System Design?
- Asked in **every senior developer interview**
- Helps you think about **big picture**, not just coding
- Real apps (Instagram, Uber, WhatsApp) need proper design to work at scale

## What Does System Design Cover?

```
1. Client & Server          - How user talks to your app
2. Database                 - Where data is stored
3. Scaling                  - Handling more users
4. Load Balancer            - Distributing traffic
5. Caching                  - Making things faster
6. CDN                      - Serving content globally
7. Message Queues           - Handling background tasks
8. Microservices            - Breaking app into small services
9. API Gateway              - Single entry point for all requests
10. Design Goals            - Scalability, Availability, Consistency
```

## Real World Analogy

```
Think of a RESTAURANT:

- Client = Customer who orders food
- Server = Waiter who takes order and brings food
- Database = Kitchen where food is prepared
- Load Balancer = Manager who assigns tables to waiters equally
- Cache = Pre-made dishes (ready to serve instantly, no cooking wait)
- CDN = Multiple branches in different cities (food closer to customer)
- Message Queue = Order slip queue in kitchen (cook processes one by one)
- Microservices = Separate stations: grill, dessert, drinks (each does one thing)
```

## How to Approach System Design Interview?

```
Step 1: CLARIFY requirements (ask questions!)
   - How many users? Read-heavy or write-heavy?
   - What features are most important?

Step 2: HIGH-LEVEL design (draw boxes and arrows)
   - Client --> Load Balancer --> Servers --> Database

Step 3: DEEP DIVE into components
   - Which database? SQL or NoSQL?
   - How to cache? Where to cache?

Step 4: ADDRESS bottlenecks
   - What if server crashes? (Add redundancy)
   - What if too much traffic? (Add load balancer)
```
