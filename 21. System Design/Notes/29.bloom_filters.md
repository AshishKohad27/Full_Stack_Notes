# 29. Bloom Filters

- Bloom Filter = a **space-efficient** data structure to check "is this item in the set?"
- Can say: **"Definitely NOT in set"** or **"PROBABLY in set"**
- Never gives false negatives, but can give false positives

## Real World Analogy

```
Security guard at a party (Bloom Filter):
  "Is this person invited?"

  Guard says "NO" → Definitely NOT invited (100% sure)
  Guard says "MAYBE YES" → Probably invited, let me double-check the full list

  Guard is fast but not 100% accurate on "yes" answers.
  But if guard says "no", it's definitely no!
```

## Why Use Bloom Filters?

```
Problem: Check if username is taken among 1 billion users

Without Bloom Filter:
  Query database for every check → SLOW (hits DB each time)

With Bloom Filter:
  Check bloom filter first (in memory, super fast):
  → "NOT in set" → Username available! (no DB query needed)
  → "MAYBE in set" → Query database to confirm

  Saves 99% of database queries!
```

## Where It's Used

```
1. CHROME BROWSER - Malicious URL check
   "Is this URL in the list of dangerous websites?"
   Bloom filter checks locally → only queries Google server if "maybe yes"

2. MEDIUM / INSTAGRAM - "Already shown" check
   "Has this user already seen this article/post?"
   Bloom filter says "definitely not seen" → show it

3. DATABASES (Cassandra, HBase)
   "Does this data exist on disk?"
   Bloom filter says "definitely not" → skip disk read (saves I/O)

4. CACHE - Avoid cache penetration
   "Does this key exist in our system?"
   Bloom filter says "no" → don't even check cache or DB
```

## Key Points
- Bloom Filter = fast, space-efficient membership test
- "Not in set" = **100% certain**
- "In set" = **probably** (small chance of false positive)
- Used to **avoid expensive lookups** (DB, disk, network)
- Trade-off: more memory = fewer false positives
