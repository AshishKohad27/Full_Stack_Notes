# 15. Proxy - Forward & Reverse

- Proxy = a **middleman** between client and server
- Two types: Forward Proxy (protects client) & Reverse Proxy (protects server)

## Forward Proxy (Client-side)

```
Client → Forward Proxy → Internet → Server

The proxy acts on BEHALF of the client.
Server doesn't know the real client.
```

### What it Does
```
1. ANONYMITY - Hides client's real IP
   Client IP: 192.168.1.1 → Proxy IP: 45.67.89.10 → Server
   Server sees: 45.67.89.10 (not your real IP)

2. CONTENT FILTERING - Block certain websites
   Company proxy: blocks facebook.com, youtube.com during work hours

3. BYPASS RESTRICTIONS - Access blocked content
   Website blocked in your country → use proxy from another country

4. CACHING - Cache frequently accessed websites
   100 employees access google.com → proxy caches it → serves from cache
```

### Real World Analogy
```
You (Client) → Your Friend (Forward Proxy) → Shop (Server)

You want to buy something but don't want the shop to know it's you.
You send your friend to buy it for you.
Shop only sees your friend, not you.
```

## Reverse Proxy (Server-side)

```
Client → Internet → Reverse Proxy → Server(s)

The proxy acts on BEHALF of the server.
Client doesn't know the real server.
```

### What it Does
```
1. LOAD BALANCING - Distribute requests to multiple servers
   Client → Nginx (reverse proxy) → Server 1, 2, 3

2. SSL TERMINATION - Handle HTTPS encryption
   Client → HTTPS → Nginx → HTTP → Backend servers
   Backend servers don't need to handle SSL!

3. CACHING - Cache responses
   Same API called 1000 times → proxy caches → serves without hitting server

4. SECURITY - Hide server details, block attacks
   Client only sees proxy IP, not actual server IPs
   DDoS attack hits proxy, not your servers

5. COMPRESSION - Compress responses before sending
   Reduces bandwidth, pages load faster
```

### Real World Analogy
```
You (Client) → Reception (Reverse Proxy) → Departments (Servers)

You go to a company. You don't go directly to departments.
Reception takes your request and directs you to the right department.
You don't know room numbers, internal structure etc.
```

## Forward vs Reverse Proxy

| Feature         | Forward Proxy            | Reverse Proxy              |
|-----------------|--------------------------|----------------------------|
| Protects        | Client                   | Server                     |
| Who uses it?    | Client/Employee          | Server/Company             |
| Hides           | Client's identity        | Server's identity          |
| Position        | Near client              | Near server                |
| Use case        | Bypass blocks, anonymity | Load balance, security     |
| Example         | VPN, corporate proxy     | Nginx, Cloudflare          |

```
FORWARD:  [Client] → [Forward Proxy] → ════════ → [Server]
                     (client side)

REVERSE:  [Client] → ════════ → [Reverse Proxy] → [Server]
                                 (server side)
```

## Nginx as Reverse Proxy (Most Popular)

```
Common Nginx setup:

Client Request → Nginx (port 80/443)
  → /api/*       → Backend Server (port 3000)
  → /images/*    → Static File Server
  → /admin/*     → Admin Server (port 5000)

Nginx handles SSL, caching, compression, routing.
Backend servers only focus on business logic.
```

## Key Points
- Forward Proxy = protects CLIENT (VPN, bypass blocks, anonymity)
- Reverse Proxy = protects SERVER (load balancing, SSL, caching, security)
- **Nginx** is the most popular reverse proxy
- Almost every large-scale system uses a reverse proxy
- Load Balancer IS a type of reverse proxy
