# 13. DNS & How the Internet Works

- DNS = **Domain Name System** = the phonebook of the internet
- Converts human-readable names to IP addresses
- You type `google.com` → DNS converts to `142.250.190.46`

## Why DNS?
```
Computers talk using IP addresses (142.250.190.46)
Humans remember names (google.com)
DNS bridges this gap!

Without DNS:
  You'd type: http://142.250.190.46 → Google
  You'd type: http://157.240.1.35   → Facebook
  Impossible to remember!

With DNS:
  You type: google.com → DNS finds the IP → connects you
```

## How DNS Works (Step by Step)

```
You type: www.google.com

Step 1: Browser Cache
  → Browser checks: "Do I already know this IP?" → If yes, done!

Step 2: OS Cache
  → OS checks its DNS cache → If found, done!

Step 3: Router Cache
  → Your WiFi router has a cache too → If found, done!

Step 4: ISP DNS Resolver (Recursive Resolver)
  → Your ISP (Jio, Airtel) checks its cache
  → If not found, starts the DNS lookup journey:

Step 5: Root Name Server
  → "I don't know google.com, but .com is handled by these servers"
  → Points to .com TLD server

Step 6: TLD (Top-Level Domain) Server
  → ".com server" says: "google.com is managed by these name servers"
  → Points to Google's Authoritative Name Server

Step 7: Authoritative Name Server
  → "google.com = 142.250.190.46" ✅ FOUND!
  → IP returned all the way back to your browser

Step 8: Browser connects to 142.250.190.46 → Google loads!
```

```
Browser → OS → Router → ISP Resolver → Root → TLD (.com) → Authoritative
                                                                   |
Browser ← OS ← Router ← ISP Resolver ←────────────────────── IP Address
```

## DNS Record Types

| Record | What It Does                  | Example                          |
|--------|-------------------------------|----------------------------------|
| A      | Domain → IPv4 address         | google.com → 142.250.190.46     |
| AAAA   | Domain → IPv6 address         | google.com → 2607:f8b0::200e   |
| CNAME  | Alias (domain → domain)       | www.google.com → google.com     |
| MX     | Mail server for domain        | google.com → mail.google.com    |
| NS     | Name server for domain        | google.com → ns1.google.com     |
| TXT    | Text info (verification)      | SPF records, domain verification|

## DNS in System Design

```
DNS Load Balancing:
  google.com → 142.250.190.46 (Server 1)
             → 142.250.190.47 (Server 2)
             → 142.250.190.48 (Server 3)

  DNS returns different IP each time = distributes traffic!
  Simple load balancing at DNS level.

DNS Failover:
  Primary:   app.example.com → 1.2.3.4 (main server)
  Secondary: app.example.com → 5.6.7.8 (backup server)

  If main server goes down, DNS switches to backup IP.
```

## What Happens When You Type a URL? (Full Picture)

```
1. You type: https://www.amazon.com/products

2. DNS Resolution
   → Browser resolves amazon.com to IP address

3. TCP Connection
   → Browser establishes connection with server (3-way handshake)

4. TLS/SSL Handshake (for HTTPS)
   → Encryption established (secure connection)

5. HTTP Request
   → Browser sends: GET /products HTTP/1.1

6. Server Processing
   → Server receives request → queries database → prepares response

7. HTTP Response
   → Server sends back HTML, CSS, JS, images

8. Browser Rendering
   → Browser parses HTML → builds DOM → applies CSS → runs JS → page displayed!
```

## Key Points
- DNS converts domain names to IP addresses
- DNS responses are cached at multiple levels (browser, OS, ISP)
- TTL (Time To Live) controls how long cache is valid
- DNS can be used for simple load balancing and failover
- DNS is hierarchical: Root → TLD → Authoritative
